# Power Measurement Run — 2026-04-19

Multi-day battery-only run to derive average current draw from Vbat delta.
Single ADC reading has ~±80 mV noise, so we need a voltage swing well above
that floor to compute a meaningful number. On a 3000 mAh LiPo at the
expected ~13 mA component budget, 3-4 days gives ~75-150 mV.

- **NOT yet live:** SX1262 `startReceiveDutyCycle()` (Phase 4 §2 — main

## Target

Come back in **3-4 days** (2026-04-22 or 2026-04-23), grab another
8-sample average, and compute:

```
avg_mA ≈ ΔV_mV / ΔV_per_mAh_on_plateau  (around 0.4 mV/mAh near 4.0 V)
       ≈ ΔV_mV × 2.5 / Δt_hours
```

Example: if end average is 3950 mV after 96 h:
- ΔV = 126 mV
- charge drawn ≈ 126 × 2.5 = 315 mAh
- average current = 315 / 96 = **~3.3 mA**

(Don't take that example as expected — it's illustrative. Actual LiPo
plateau slope varies with cell age and the flat region between 4.0 V and
3.95 V is notoriously unreliable for SoC estimation, so the result is
still ±30 %.)

## Invalidation conditions — restart the run if ANY occur

- Any `./flash.sh` operation (reflash = USB = charging)
- Device physically plugged into USB for any reason (even 5 min charges)
- Unexpected cold reboots beyond a handful (each burns a ~30 s radio init
  at elevated current; too many skews the average)
- Battery swap / external power source connected
- Thread / iot_log silence gap > 12 h with no STATUS lines captured —
  means the end-of-run Vbat can't be compared to start cleanly

## What to grep at the end

```
grep "STATUS:" /tmp/iot_e2e.log | sed 's/\x1b\[[0-9;]*m//g' | \
  awk '/<YYYY-MM-DD HH:MM range>/ { extract vbat }'
```

Plus check for reboots (uptime rewinds) and iot_log silence gaps in the
interval, both of which affect the integration.
